# TutoTuto 新UI設計 ディスカッションメモ

日付: 2026-03-18

---

## 背景・課題

現状のTutoTutoは「PDFにペン書き → 採点対象を囲う → 採点」で終わり。
中学・高校レベルになると採点後に「なぜ間違えた？」「もっと詳しく」「類題を出して」など
AIへの深掘りニーズが出てくる。

---

## 新しいインタラクション設計

### 統一操作：「囲う → 書く → 送る」

- PDFでもAI回答でも同じ操作で深掘りできる
- 操作対象が何であれUIは変わらない

### フロー

```
対象を囲う → テキスト入力 → 送信 → 新パネルが開く → また囲う → ...
```

**採点フロー：**
```
PDFの解答範囲を囲う → 解答/コメントを書く → 採点依頼 → 採点結果パネル
```

**深掘りフロー：**
```
採点結果パネルの気になる箇所を囲う → 質問を書く → 送信 → AI回答パネル
→ AI回答パネルの気になる箇所を囲う → 質問を書く → 送信 → さらにAI回答パネル
→ ...（無限に深掘り可能）
```

---

## 画面遷移設計

### パネル表示

- 1パネルがほぼ全画面（95%程度）を占有
- 新パネルが右からスライドインして前を覆う
- HistoryFinderのような「複数パネルが横並び」ではなく全画面切り替え

### ナビゲーション

- **パンくずをヘッダに配置**（HistoryFinderと同じ方式）
- 例: `[PDF] > [採点結果] > [なぜ間違えた？] > [類題]`
- パンくずをクリックすると任意の深さに戻れる

### スワイプについて

- スワイプはPDFページめくり・パン操作に使用するため、パネル遷移には使わない
- 戻る操作はヘッダのパンくずのみ

---

## AI会話のコンテキスト保持方針

- **採点画像は最初のパネルのみ保持**（以降はテキスト化されたコンテキストで継続）
- 画面遷移設計とは独立しており、後からAまたはBに変更可能

**選択肢：**
- A: 起点（採点画像）を常に保持 → トークン増・文脈維持
- B: 会話履歴のみ保持（画像は最初だけ）→ 軽量・実用上十分（当面はB）

---

## プロトタイプ方針

### 含めるもの
- 左端にモックPDFパネル（ダミー画像でOK）
- ドラッグで範囲選択
- テキスト入力 → 送信
- 右に採点結果パネルがスライドイン
- 採点結果パネルでも囲う → 質問 → 右にAI回答パネル
- ヘッダにパンくず表示・クリックで戻る

### 省略するもの
- 実際のAI API呼び出し（固定ダミー回答）
- Firebase認証・Stripe等
